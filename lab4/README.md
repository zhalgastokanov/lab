# Лабораторная работа №4: Улучшение качества модели

## Дисциплина

Анализ данных и искусственный интеллект

## Цель работы

Научиться повышать качество моделей машинного обучения, выполнять подбор гиперпараметров, сравнивать разные алгоритмы и анализировать переобучение.

## Предметная область

Датасет отражает поведение пользователей смартфонов, включая:

- количество времени, проведённого за телефоном и в соцсетях,
- рабочую продуктивность,
- сон и уровень стресса,
- использование приложений и потребление кофеина.

Цель анализа: улучшить качество модели, которая предсказывает уровень рабочей продуктивности пользователя на основе его цифровых привычек и образа жизни.

## Исходные данные

- Файл: `data/raw/dataset_final_onehot.csv`
- Источник: подготовленный датасет из лабораторной работы №3
- Целевая переменная: `Work_Productivity_Score`
- Тип задачи: регрессия

## Используемые библиотеки

- pandas
- numpy
- matplotlib
- seaborn
- scikit-learn

## Используемые модели

1. **Linear Regression**
   - Базовая модель из лабораторной работы №3.
   - Используется для сравнения с более сложными алгоритмами.

2. **Decision Tree Regressor**
   - Альтернативная модель.
   - Позволяет находить более сложные нелинейные зависимости между признаками.

3. **Decision Tree Regressor + GridSearchCV**
   - Улучшенная версия дерева решений.
   - Используется подбор гиперпараметров для повышения качества модели и уменьшения переобучения.

## Шаги выполнения

1. **Загрузка подготовленного датасета**
   - Загружен файл `dataset_final_onehot.csv`.
   - Выполнен первичный просмотр данных.

2. **Подготовка признаков и целевой переменной**
   - Целевая переменная: `Work_Productivity_Score`.
   - Остальные признаки используются как входные данные для модели.

3. **Разделение данных на обучающую и тестовую выборки**
   - Использован `train_test_split` из `sklearn.model_selection`.
   - Размер тестовой выборки: 20%.
   - Для повторяемости результата указан `random_state=42`.

4. **Обучение базовой модели**
   - Обучена модель `LinearRegression`.
   - Рассчитаны метрики качества на обучающей и тестовой выборках.

5. **Обучение альтернативной модели**
   - Обучена модель `DecisionTreeRegressor`.
   - Выполнено сравнение качества с линейной регрессией.

6. **Подбор гиперпараметров**
   - Использован `GridSearchCV`.
   - Подбирались параметры:
     - `max_depth`
     - `min_samples_split`
     - `min_samples_leaf`
   - Оценка выполнялась по метрике `R2`.

7. **Обучение улучшенной модели**
   - Использована лучшая модель, найденная через `GridSearchCV`.
   - Рассчитаны метрики качества на train и test.

8. **Сравнение моделей**
   - Создана таблица сравнения моделей.
   - Использованы метрики:
     - `MAE`
     - `MSE`
     - `R2`
   - Сравнение выполнено отдельно для обучающей и тестовой выборок.

9. **Анализ переобучения**
   - Рассчитана разница между `Train R2` и `Test R2`.
   - Рассчитана разница между `Test MAE` и `Train MAE`.
   - Определено, какая модель склонна к переобучению.

10. **Визуальный анализ**

- Построен график сравнения моделей по `Test R2`.
- Построен график сравнения моделей по `Test MAE`.

11. **Формулировка выводов**

- Базовая линейная модель используется как начальная точка сравнения.
- Обычное дерево решений может переобучаться.
- Подбор гиперпараметров позволяет ограничить сложность дерева.
- Лучшая модель выбирается по более высокому `Test R2` и меньшему `Test MAE`.

## Метрики качества

Для оценки моделей использовались следующие метрики:

- **MAE** — средняя абсолютная ошибка.
- **MSE** — среднеквадратичная ошибка.
- **R2** — коэффициент детерминации, показывающий, насколько хорошо модель объясняет данные.

Была выполнена настройка гиперпараметров с помощью `GridSearchCV`, а также проведён анализ переобучения.
